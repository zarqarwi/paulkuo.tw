## 可能原因與修復方向

### 症狀一：定位飄移（GPS 點跑到不合理的地方）

**最可能原因**：LINE WebView 的 `Geolocation API` 在初始化時會給出精度很差的點（accuracy 可能高達數百或數千公尺），這個點如果沒被過濾就直接上傳。

**修復方向**：
```typescript
// 上傳前加 accuracy 門檻過濾
if (position.coords.accuracy > 100) {
  // accuracy 超過 100 公尺的點，視為不可靠，不上傳
  return;
}
```

確認前端目前有沒有這個判斷；如果沒有，加上去。

**備選原因**：LINE WebView 在背景切換或 app 恢復時，`watchPosition` 可能給出快取舊點。加上 `maximumAge: 0` 強制要求新鮮定位：
```typescript
navigator.geolocation.watchPosition(success, error, {
  enableHighAccuracy: true,
  timeout: 10000,
  maximumAge: 0  // 不接受快取位置
});
```

---

### 症狀二：繞圈移動但只顯示直線

**最可能原因一**：server-side clustering 把兩個時間相近但距離較近的點合併，丟棄了中間的路徑節點。

**修復方向**：調整 clustering 的距離門檻（如果 `clusterRadius` 太大，把正常走路的點都合併掉）。

**最可能原因二**：前端只上傳「打卡點」（用戶手動按按鈕的那一刻），而不是持續記錄路徑。如果只有兩個打卡點，就一定是直線。

**確認方式**：查前端是 `watchPosition`（持續追蹤）還是只在打卡時 `getCurrentPosition`（一次性定位）。如果是後者，路徑直線是設計決策，不是 bug，應該跟 Paul 確認預期行為。
