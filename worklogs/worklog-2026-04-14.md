- osascript 跑指令內含特殊字元（!、$ 等）時 bash quoting 很容易爆。用 `do shell script` 送 bash 時建議「指令越短越好」，複雜邏輯包進 script 檔案再呼叫
- @_davideast/stitch-mcp 這個 npm 包名字很誤導 — 它不是 MCP server，是「OAuth/API key 設定助手」，跑 `init` 子指令會印出真正要設給 Claude client 的 config（指向 https://stitch.googleapis.com/mcp 的 HTTP MCP）。直接 `npx @_davideast/stitch-mcp` 沒帶子指令會印 help 然後 exit，被 MCP client 誤認為「server 啟動後立刻死掉」
- Claude Desktop / Cowork 對於不支援的 MCP entry 格式（如 type:http）會「靜默跳過」，連 failed 狀態都不顯示。診斷方式：在 Settings → Developer 看設定的 MCP 是否「整條消失」
