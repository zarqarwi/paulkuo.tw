## 怎麼進入後台

**網址**：`/projects/formosa-esg-2026/dashboard/`

打開後需要輸入管理密碼。系統有三層權限，不同密碼看到不同內容：

| 角色 | 密碼 | 可見範圍 |
|------|------|----------|
| Owner（站長） | `gf-admin` | 全部功能 |
| Manager（管理者） | `gf-manager` | LINE 額度、推播、參與者列表、地圖、回饋紀錄 + 志工可見區塊 |
| Volunteer（志工） | `g-formosa` | 碳減量、等級分布、問卷洞察、活動時間軸、提交回饋 |

一般管理者用 `gf-manager` 登入。

如果只是要展示給別人看，可以在網址後面加 `?demo=1`，會進入 Demo 模式——資料唯讀，不需要密碼。

## 一、數據儀表板

進入後台後，第一眼看到的是六個數字：

- **總參與人數** — 有多少人透過 LINE 登入了
- **今日活躍** — 今天有打卡或交資料的人數
- **總打卡次數** — 所有人加起來打了幾次卡
- **總步行公里** — 所有人走了多少公里
- **碳足跡** — 大家總共產生多少碳排放（單位是 kg）
- **碳節省量** — 大家因為用走的，省下了多少碳

### 地圖

地圖可以用三種方式看：

**熱力圖** — 顏色越深代表越多人在那邊打卡，適合一眼掃過人潮分佈。

**標記點** — 每個打卡的地方都會標一個點。

**聚類模式** — 把附近的點合成一團顯示，人多的時候比較清楚。

地圖上還有兩個標示：
- 🟢 **前鋒** — 走最前面的人在哪
- 🔴 **尾巴** — 走最後面的人在哪

上方會顯示「白沙屯→北港」的進度百分比，還有前鋒跟尾巴之間的距離，讓你知道隊伍拉多長。

### 分析面板

**碳足跡面板** — 碳排放跟碳節省的統計。系統怎麼算的：走路或騎腳踏車（時速 15 以下）算零排放，超過時速 15 的就當作搭車來估算。這個數字是拿來提醒大家減碳用的，不是精確的碳盤查。

**等級分布** — 9 個等級各有多少人，看得出大家走到什麼階段。

**問卷洞察** — 問卷 Q1 到 Q6 的統計結果。

**活動時間軸** — 什麼時間打卡最多，可以按日期或時段看。

## 二、推播通知

### 推播模式

系統支援四種格式：

**文字訊息** — 寫標題跟內容，最基本的方式。

**圖片訊息** — 貼上圖片 URL，直接發一張圖。輸入 URL 後會有即時預覽，確認圖片正確再發。

**圖文組合** — 同時發一張圖片和一段文字說明，適合活動公告搭配照片。

**URL 預覽** — 發送帶有連結預覽的訊息，適合分享網頁。

### 手動發推播

1. 選推播對象：全部 / 香客 / 志工 / 管理者
2. 選推播模式（文字 / 圖片 / 圖文 / URL）
3. 填寫內容
4. 按「發送推播」

按下去就直接送到香客的 LINE，沒有「確認」按鈕，所以發之前請再看一次。

「暫離中」跟「已完成」的人不會收到，系統會自動跳過。

### 自動推播

活動期間（4/12 到 4/20），系統每天會在 6:00、9:00、12:00、15:00、18:00 自動推播，內容是進度更新、鼓勵、安全提醒。

## 三、參與者管理

後台有一個使用者列表，可以看每個人的名稱、打卡次數、碳排數據、最後活躍時間、角色。上方有搜尋欄，點表頭可以排序。

### 角色

| 角色 | 差別在哪 |
|------|----------|
| 香客（預設） | 一般使用者，可以打卡、填問卷、分享成就 |
| 志工 | 可進後台查看碳減量、等級分布、問卷洞察、活動時間軸、回饋紀錄。推播可單獨發給志工 |
| 管理者 | 可看後台全部區塊，包含推播發送和使用者列表 |

要改角色的話，目前要透過 API 操作，請找工程師協助。

### 使用者狀態

| 狀態 | 代表什麼 |
|------|----------|
| 活躍中 | 正在參與進香，會收到推播 |
| 暫離中 | 香客按了「暫離」，推播暫停、定位停止。重新打開頁面就會自動恢復 |
| 已完成 | 香客按了「完成進香」，成就卡定版，不再收到推播 |

活動結束時，系統會把所有還在「活躍」或「暫離」的人統一結算為「已完成」。

## 四、活動控制

活動有三種狀態：

| 狀態 | 效果 |
|------|------|
| 進行中 | 一切正常運作 |
| 暫停 | 香客端的打卡等功能暫時關閉 |
| 結束 | 活動結束，所有功能關閉 |

什麼時候用暫停：颱風天、系統維護、突發狀況。
什麼時候用結束：活動正式結束。

切錯了沒關係，可以再改回「進行中」。目前切換要透過 API 操作，請找工程師協助。

## 六、碳足跡怎麼算的

系統用兩個簡單的規則來估算碳排放：

**規則一：時速 15 以下 = 零排放。** 走路、慢跑、腳踏車都算在這。

**規則二：時速超過 15 = 搭車。** 不管是搭公車、騎機車還是開車，統一用大眾交通工具人均係數（0.12013 kg CO₂e/person·km，來源：ecoinvent「market for transport, regular bus」）來估。

這個速度是系統從 GPS 打卡的軌跡自動算出來的，香客不需要自己填。

另外，問卷中填寫的住宿方式（Q5）和到場交通方式（Q3）也會納入碳足跡估算。

「碳節省量」的意思是：如果全程都搭車，碳排會是多少？扣掉實際的碳排，差額就是省下來的。

> 這些數字是拿來提醒大家注意減碳的，不是正式的碳盤查報告。數據來源是 Ecoinvent 3.10 跟 SSBTi 善足跡。

## 七、個人資料保護

本系統蒐集的個人資料受台灣《個人資料保護法》規範。

### 蒐集了什麼

| 資料 | 是否必要 |
|------|----------|
| LINE 帳號（用戶 ID、名稱、頭像） | 必要 |
| GPS 座標與移動軌跡 | 必要 |
| 照片裡的 GPS 座標（照片本身不會上傳） | 選填 |
| 問卷回答（善行、交通、環保行為、住宿、用餐、參與身分等） | 必要 |
| 手機號碼（抽獎通知用） | 選填 |

### 同意機制

香客在 LINE 登入之後、填問卷之前，一定要先讀過隱私權聲明並按「我同意」。沒同意就不能用打卡功能。這一步不能跳過、不能預設勾選。

### 資料保留

- 活動期間：正常使用
- 結束後 6 個月內：做 ESG 報告、抽獎聯繫、商家驗證
- 6 個月後：可辨識個人的資料刪除或匿名化，只留統計數據

### 管理者要特別注意的事

- **不要把含個資的資料匯出到不安全的地方**，例如公開的 Google Sheet
- **不要把後台密碼給非相關人員**，三組密碼各有其權限範圍，請依角色分配
- 如果有人要求刪除他的資料，要配合處理
- 合作商家只能透過掃 QR Code 看到有限資訊（等級、進度），**不能**給他們看完整資料庫
- 手機號碼只能用在抽獎通知，**不能**拿去做行銷或給第三方

## 八、系統安全防護

本系統在設計上已將資料安全納入考量。以下說明目前的防護機制，讓管理者了解系統是如何保護參與者的資料。

### 傳輸安全

全站使用 HTTPS 加密傳輸，部署在 Cloudflare 全球邊緣網路上。所有從香客手機到伺服器之間的資料（GPS 座標、問卷回答、個人資料）都經過加密，第三方無法在傳輸過程中攔截或竄改。

### 身份驗證

系統採用 LINE Login（OAuth 2.0）做身份識別，不自行管理帳號密碼，降低密碼外洩的風險。香客不需要另外註冊帳號，登入流程由 LINE 官方處理。

管理後台需要獨立的管理員密碼才能進入，一般香客無法存取後台資料。三組管理密碼以加密方式儲存在 Cloudflare 的安全環境變數中（`FORMOSA_ADMIN_TOKEN` / `MANAGER_TOKEN` / `VOLUNTEER_TOKEN`），不會出現在程式碼裡。

### 資料庫防護

所有資料庫查詢使用參數化語法（Parameterized Queries），防止 SQL 注入攻擊。即使有人嘗試在輸入欄位塞入惡意指令，系統也不會執行。

GPS 座標設有範圍驗證，不在合理範圍內的座標會被拒絕，防止假資料污染。

### 防濫用機制

打卡功能設有頻率限制：每人每 60 秒最多 5 次打卡。超過的會被系統擋下，防止有人用程式灌假資料。問卷提交也有類似的保護機制。

### 照片隱私

照片完全不會上傳到伺服器。系統只在香客的手機上讀取照片裡的 GPS 座標（EXIF 資料），讀完就丟掉，照片本身不離開手機。這代表即使伺服器被入侵，也不會外洩任何照片。

### 資料備份

系統設有多層備份機制：

**即時緩衝**：GPS 打卡資料會先寫入高速緩存（Cloudflare KV），再定時批次寫入主資料庫。如果主資料庫暫時寫入失敗，緩存會自動延長保留時間，確保資料不遺失。

**資料庫備份**：Cloudflare D1 資料庫內建 Time Travel 功能，可以回溯過去 30 天的任意時間點。如果資料被誤刪或損毀，可以還原到出事前的狀態。

**離線備份**：系統每天自動匯出一份完整資料備份到管理者的本機電腦，包含使用者、問卷、GPS 軌跡等所有資料表。備份檔案保留在本地，不上傳到任何第三方服務。

### 隱私權同意流程

香客在 LINE 登入後、使用任何功能之前，必須閱讀完整的隱私權聲明並主動按下「我同意」。沒有預設勾選、沒有跳過機制。同意紀錄會存在伺服器上，可供稽核。

### 管理者的安全責任

身為管理者，請特別注意以下事項：

- **三組密碼不要分享**給非相關人員，更不要用通訊軟體傳送
- **不要在公開場合展示後台**，如果需要展示請用 Demo 模式（網址加 `?demo=1`）
- **匯出的資料要妥善保管**，不要放在公開的雲端空間或共用資料夾
- **手機號碼等個資只限活動用途**，不得用於行銷或提供給第三方
- 如果發現異常（例如不明的大量打卡、資料異常），請立即通知工程師

### 安全事件處理

如果發生資料外洩或安全事件：

1. 立即通知工程師，暫停受影響的功能
2. 評估影響範圍（哪些資料、多少人受影響）
3. 如涉及個人資料，依《個人資料保護法》通知當事人
4. 記錄事件經過，作為後續改善依據

## 九、QR Code 與成就卡

香客達到以下三個條件後，系統才會產生成就卡：

1. **打卡至少 3 次**
2. **完成問卷**
3. **有留下電話號碼**

三個都要達標，缺一不可。沒達標的香客不會看到成就卡選項。

成就卡上會顯示等級公仔、步行距離、打卡次數、碳足跡、移動方式分佈，還有一組 QR Code。掃描 QR Code 會連到那個人的個人足跡頁面，可以看到完整的進香紀錄。

不一定要走完全程才能拿到成就卡 — 只要達標就有，等級會反映實際的參與程度。

這個可以用在：
- 合作商家確認香客的參與程度，再決定要不要給贈品
- 當作 ESG 報告的參與證明
- 香客分享給朋友看

## 十、部署與維運

> 詳細流程圖請看 **[管理者操作流程圖](/projects/formosa-esg-2026/guide/admin-flow/)**

### 前端部署

```bash
cd ~/Desktop/01_專案進行中/paulkuo.tw
npm run build && wrangler deploy
```

CDN 快取 max-age=3600，新部署最多等 1 小時生效。急的話到 Cloudflare Dashboard 手動清 cache。

### Worker API 部署

```bash
cd ~/Desktop/01_專案進行中/paulkuo.tw/worker
wrangler deploy --config wrangler.toml
```

一定要帶 `--config`。根目錄有 `wrangler.jsonc` 會覆蓋設定，不帶的話會部署到錯誤的 Worker。

### 部署後驗證清單

每次部署後確認這四件事：

1. 開 `/tracker/` → LIFF 登入能用
2. 開 DevTools → Worker API 回 200
3. 登後台 → KPI 數字有更新
4. 傳「打卡」給 LINE Bot → 連結可開

### 活動期間每日監控

活動期間（4/12–4/20）每天要看：

- **Cloudflare Dashboard** — Workers 請求量、錯誤率、D1 讀寫、KV 用量
- **管理後台地圖** — 前鋒/尾巴位置合理、無異常座標
- **LINE Messaging API** — 推播用量、送達率
- **每日備份** — 確認離線備份有跑、檔案大小正常

### 異常處理

| 狀況 | 處理步驟 |
|------|----------|
| API 大量 500 | 查 Workers Logs → 確認 D1 連線 → 必要時切「暫停」 |
| 打卡資料異常 | 查 KV Buffer 堆積 → 手動觸發 flush → 檢查 rate limit |
| LINE Bot 不回 | 確認 webhook URL → 檢查 API 額度 → 重啟 Worker |
| 資料要還原 | D1 Time Travel → 回溯 30 天內任意時間點 |

### 活動結束收尾

1. 活動切「結束」→ 所有人成就卡統一定版
2. 匯出全部資料（使用者、GPS、問卷）
3. 整理碳足跡數據做 ESG 報告
4. 個資保留 6 個月後匿名化或刪除

### 已知技術陷阱

- 根目錄 `wrangler.jsonc` 覆蓋問題 → Worker 部署一定帶 `--config`
- CDN 快取 1 小時 → 急的話手動清
- LINE in-app browser 跟 Safari 的 localStorage 是隔離的
- cron 每 10 分鐘 git stash/pop → commit + push 要原子操作
