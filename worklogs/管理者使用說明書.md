## 一、數據儀表板

進入後台後，首先看到的是 KPI 總覽：

| 指標 | 說明 |
|------|------|
| 總參與人數 | 已透過 LINE 登入的使用者數 |
| 今日活躍 | 今天有打卡或提交資料的人數 |
| 總打卡次數 | 所有人的打卡總次數 |
| 總步行公里 | 所有人累計的步行距離 |
| 碳足跡 | 全體碳排放統計（kg CO₂e），由 GPS 軌跡速度自動推算交通方式 |
| 碳節省量 | 全體步行省下的碳排放（相比開車） |

### 即時路徑地圖

地圖有三種模式可以切換：

**熱力圖** — 用顏色深淺顯示 GPS 打卡的密度分佈，一眼看出人潮集中在哪。

**標記點** — 每個打卡點顯示為一個標記。

**聚類模式** — 用 ~2km 網格把附近的打卡點合在一起顯示，適合人數很多的時候看整體分佈。

地圖上還會標示兩個特殊位置：
- 🟢 **前鋒** — 目前最北端（最前面）的打卡者位置
- 🔴 **尾巴** — 目前最南端（最後面）的打卡者位置

另外還有：
- **白沙屯→北港進度條** — 顯示整體隊伍走到哪了，用百分比呈現
- **展開距離** — 前鋒和尾巴之間的距離（km），看得出隊伍拉多長

### 數據面板

儀表板下方有幾個分析面板：

**碳足跡面板** — 累積碳排放與碳節省量統計。交通方式由 GPS 軌跡的移動速度自動推算（步行 < 8 km/h、腳踏車 8-20 km/h、機車 20-40 km/h、汽車 40-80 km/h、高鐵/火車 > 80 km/h），並顯示各交通方式的公里數分佈比例。

**等級分布面板** — 9 個等級各有多少人，看得出大部分香客走到哪個階段。

**問卷洞察面板** — Q1～Q6 的問卷統計結果，包含角色分佈（主辦方 vs 參與人）的直方圖。

**活動時間軸面板** — 打卡和提交活動的分佈，可以按日期或時段查看，了解什麼時間最活躍。

## 二、推播通知

後台可以直接發推播訊息到香客的 LINE。

### 手動推播

1. 選擇推播對象：全部 / 香客 / 志工 / 管理者
2. 填寫標題和內容
3. 按「發送推播」

按下去就會立刻送出，送出前請再確認一次內容。

注意：狀態為「暫離中」或「已完成」的香客不會收到推播，系統會自動排除。

### 排程推播

活動期間（4/12～4/20），系統會在每天的固定時段自動推播提醒：

- 6:00 AM
- 9:00 AM
- 12:00 PM
- 3:00 PM
- 6:00 PM

推播內容包含進度更新、鼓勵、安全提醒等。

## 三、參與者管理

後台有一個使用者列表，可以看到所有已註冊的參與者。

每一列會顯示：名稱、打卡次數、碳排數據、最後活躍時間、角色（participant / volunteer / admin）。

**搜尋** — 上方有搜尋欄，輸入關鍵字即時過濾。

**排序** — 點表頭可以依打卡次數、碳排、活躍時間排序。

**角色指派** — 可以把某個使用者的角色改為志工（volunteer）或管理者（admin）。目前透過 API 操作（`/api/formosa/admin/roles` POST）。

角色的差異：
- **participant**（香客）— 預設角色，使用打卡、問卷、分享等功能
- **volunteer**（志工）— 目前功能跟香客相同，但推播可以獨立針對志工發送，未來會擴充專屬功能
- **admin**（管理者）— 可以進入 Dashboard

使用者狀態：
- **活躍中** — 正在參與進香，會收到推播
- **暫離中** — 香客按了「暫離」，推播暫停、GPS 停止。重新打開 tracker 會自動恢復為活躍
- **已完成** — 香客按了「完成進香」，成就卡已定版，不再收到推播。活動結束時，系統會統一將所有仍活躍/暫離的香客結算為已完成

## 四、活動控制

活動有三種狀態：

| 狀態 | 說明 |
|------|------|
| **active**（進行中） | 正常運作，所有功能開放 |
| **paused**（暫停） | 打卡等 API 回傳 403，香客無法操作 |
| **ended**（結束） | 活動結束，功能關閉 |

切換方式：透過 API `/api/formosa/admin/status` POST 修改狀態。

暫停通常用在：天氣惡劣需要暫停追蹤、系統維護、或突發狀況。

結束用在：活動正式結束時。結束後香客端的打卡功能會被禁用。

如果誤操作，可以再切回 active 恢復。

## 技術備忘

這些是管理者可能需要知道的技術細節：

**認證方式** — 目前是 token-based（`X-Admin-Token`），不是帳號密碼制。

**碳排係數** — 依據 SSBTi 善足跡 / Ecoinvent 3.10 LCA 標準計算。交通碳排由 GPS 軌跡移動速度自動推算交通方式後套用對應係數（步行 0、腳踏車 0.01220、機車 0.13734、汽車 0.30479、高鐵 0.07487 kg CO₂e/km）。非交通碳排（住宿 12.5 kg/晚、瓶裝水 0.10974 kg/瓶）由香客每日回報。

**CDN 快取** — 前端部署後最多需要 1 小時才會全面生效（max-age=3600）。

**QR Code 驗證** — 成就卡上的 QR Code 會連到個人頁 `?u={userId}`，可作為身分驗證用途。合作商家（例如全家）可以掃描 QR Code 確認香客身分和完成進度，用於全程參與者的贈品兌換。

**手機號碼與抽獎** — 問卷中的手機號碼欄位（選填）是為了日後抽獎活動通知用。管理者可以從使用者列表篩選全程完成的香客，配合手機號碼進行抽獎聯繫。

**路徑圖作為參與證明** — 每位香客的 GPS 路徑紀錄和打卡軌跡，搭配等級、碳足跡和交通方式明細，構成一份完整的參與證明。這些資料可用於：碳足跡 LCA 計算的佐證（含 GPS 速度推算的交通方式分佈）、全程參與者的身分驗證、合作商家的兌換依據。

**部署** — 需要在本機執行 wrangler deploy，Cowork / Code session 無法直接部署。

## 個人資料保護（管理者責任）

本系統蒐集的個人資料受台灣《個人資料保護法》規範，管理者需要知道以下事項：

### 我們蒐集了什麼

| 資料類別 | 項目 | 性質 |
|----------|------|------|
| LINE 帳號資訊 | 用戶 ID、顯示名稱、大頭照 URL | 必要 |
| GPS 定位資料 | 即時座標、打卡座標、移動軌跡 | 必要 |
| 照片 EXIF 座標 | 照片中的 GPS 座標（照片本身不上傳） | 選填 |
| 問卷回答 | 善行、感動經驗、參與身分等 | 必要 |
| 每日善足跡回報 | 到場交通、瓶裝水、住宿（進香途中交通方式由 GPS 自動推算，不需手動填報） | 選填 |
| 手機號碼 | 用於抽獎通知 | 選填 |

### 同意機制

香客在 LINE 登入後、填問卷前，必須先閱讀隱私權聲明並按下「我同意」。沒有同意的人無法使用打卡功能。這個步驟不能跳過、不能預設勾選。

### 資料保留期限

- 活動期間：正常使用
- 活動結束後 6 個月內：ESG 報告彙整、抽獎聯繫、商家兌換驗證
- 6 個月後：個人可識別資料刪除或匿名化，僅保留匿名統計資料

### 管理者注意事項

- **不要匯出**含個資的資料到不安全的地方（例如公開的 Google Sheet）
- **不要分享** admin token 給非管理者
- 如收到香客要求刪除資料的請求（email），需配合處理
- 合作商家只能透過掃 QR Code 查看有限資訊（等級、進度），**不能**給他們存取完整資料庫的權限
- 手機號碼僅限抽獎通知使用，**不得**轉作行銷或轉交第三方

完整隱私權聲明請見系統內的「隱私權聲明」頁面。

## 志工角色（預留功能）

目前資料庫已經預留了 `role = 'volunteer'` 欄位，管理者可以指派。

現階段志工的使用體驗跟香客完全相同，差別只有：
- 推播可以單獨針對志工角色發送
- 後台使用者列表可以看到誰是志工

未來規劃可擴充的功能：志工專屬後台、任務派發、打卡地點驗證等。目前尚未開發。
