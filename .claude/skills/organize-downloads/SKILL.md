# 下載資料夾整理工作流程

Paul 的電腦有一套固定的資料夾分類系統，所有散落的檔案最終都要歸入這個系統。
**下載資料夾不留任何檔案，全部搬到桌面的分類資料夾。**

## 桌面資料夾架構（目的地）

| 資料夾 | 放什麼 |
|--------|--------|
| `00_收件匣` | 待處理的生活文件（機票、住宿、收據），以及暫時無法判斷歸類的檔案 |
| `01_專案進行中` | 目前進行中的專案，每個專案一個子資料夾 |
| `02_參考資料` | 技術文件、研究筆記（.md）、逐字稿、翻譯稿、產業報告、對話紀錄、腳本程式 |
| `03_媒體素材` | 圖片、影片、截圖、錄音，依子資料夾分開 |
| `04_歸檔` | 已完成或不再活躍的備份、舊版檔案、壓縮檔（.zip） |
| `05_待刪除` | 敏感憑證（client_secret、API key 的 JSON）、暫時性開發檔、確認不需要的重複檔 |

## 分類細則

### 01_專案進行中

Paul 手上常見的專案子資料夾（會隨時間增減，整理前先用 `ls` 確認目前有哪些）：

- `SDTI_BD業務開發` — 業務開發、客戶提案（CyberSolutions、City OS、Sustainable City 系列）
- `SDTI_內部` — 公司內部文件（薪酬、出差報告、工作日誌）
- `CLGM建材` — 佳龍科技/佳大建材相關（出貨、客戶開發、修秘產品）
- `LINE_WORKS代理` — LINE WORKS 代理商評估
- `paulkuo.tw` — 個人品牌、Medium 文章、寫作相關
- 其他專案：如果檔案明顯屬於某個專案但桌面上還沒有對應資料夾，就新建一個

判斷原則：合約、意向書、簡報、企劃書 → 看內容歸到對應專案。

### 02_參考資料

- 技術文件、研究筆記（.md 為主）
- 逐字稿（transcript）、翻譯稿（translated）
- 產業簡報、報告（非進行中專案的）
- LINE / 通訊軟體匯出的對話紀錄
- 自動化計畫、架構文件
- 學術 / 哲學相關筆記

### 03_媒體素材

依子資料夾分類：

- `影片/` → .mp4, .mov
- `圖片/` → .png, .jpg, .heic（非截圖的圖片）
- `截圖/` → 檔名含「截圖」的圖片
- `錄音檔/` → .mp3, .m4a

### 00_收件匣

- 機票、住宿確認、收據等待處理的生活文件
- 發票、對帳單、薪資單（需要 Paul 後續處理的財務文件）
- 短期內需要處理但不確定歸類的檔案

### 04_歸檔

- 舊版網站 HTML 檔案
- 已匯出的壓縮檔（.zip, .tar.gz）
- 不再活躍但需要保留的備份
- Cowork 產出的 files 系列資料夾

### 05_待刪除

- 敏感憑證檔案（client_secret、API key 等 JSON）
- 暫時性的開發檔案（已用過的 .jsx、.yml）
- 確認不需要的重複檔案

## 執行流程

1. **掛載資料夾** — 確認桌面（Desktop）和下載（Downloads）都已掛載，沒有的話用 `request_cowork_directory` 請求
2. **確認桌面結構** — `ls` 桌面和 `01_專案進行中` 的子資料夾，了解目前有哪些專案
3. **列出待整理檔案** — `ls -la` 下載資料夾，全部列出
4. **逐一分類** — 依照上述規則，把每個檔案搬到桌面對應資料夾
5. **跨磁碟注意** — Downloads 和 Desktop 可能是不同掛載點，`mv` 跨裝置會失敗。改用 `cp` 複製後再 `rm` 原檔（需先請求刪除權限）
6. **清空下載資料夾** — 搬完後確認 Downloads 內已無檔案
7. **回報結果** — 告訴 Paul 每個類別搬了多少檔案、搬到哪裡

## 注意事項

- 遇到無法判斷的檔案，放到 `00_收件匣`，不要自己發明新的分類資料夾
- 不要在 Downloads 裡面建子資料夾做分類 — 所有東西都搬到桌面
- 桌面上如果有散落的檔案，也一併整理進 00~05 系統
- 處理前先問 Paul 有沒有特殊需求，但如果他說「你先整理」就直接按規則做
