# Paul Kuo 內容架構 — 寫作腳本 v1

> 一篇文章，同時服務三個讀者：人的靈魂、人的理智、機器的解析器。

---

## 核心哲學：道成肉身的敘事結構

人類文明中最具穿透力的文本，無論東西方，都遵循同一個深層文法：

**從具體 → 到抽象 → 再回到具體（但讀者已經不一樣了）**

莊子用寓言起手，展開形上學，再落回「庖丁解牛」的刀刃上。
蒙田從個人軼事出發，進入哲學思辨，最後回到一個更清醒的日常。
Paul Graham 從一個反直覺的觀察開始，建構論證，結尾回到那個觀察但意義已經翻轉。

你的神學訓練給了你最精準的名字：**道成肉身**。
道（抽象的秩序）必須成為肉身（具體的場景），否則它只是論述。

這就是你每一篇文章的底層結構。

## 六幕結構

每篇文章由六個段落單元組成。不需要每一幕都很長，有些只需要一兩句話。
重點是節奏，不是長度。

### 第一幕：入場 — The Threshold

**功能：** 把讀者拉進一個具體的場景或瞬間。
**語法：** 感官的、現場的、時態明確的。
**長度：** 1-3 段。

這不是「前言」或「背景介紹」。這是一扇門。讀者推開門，看見一個畫面。

```
【範例】
那天在大阪的回收工廠，我看著一條產線把廢棄的電路板
送進酸洗槽。工人戴著全罩式面具，動作很慢。
空氣裡有一種甜膩的金屬味。

我突然想到，這條產線處理的不是垃圾，是秩序的殘骸。
```

```
【範例】
Claude 給我的第一個回答是錯的。不是事實錯誤，
是框架錯誤——它用了一個我三天前已經否決的假設。
我盯著螢幕想：這就是為什麼多模型辯論比單一模型重要。
```

**給 AI Agent 的信號：** 第一幕通常包含「場景設定」和「問題的種子」。
如果你在解析這篇文章，第一幕的最後一句話往往是全文的暗示性論點。

---

### 第二幕：張力 — The Tension

**功能：** 揭示「為什麼這件事比你想的更重要」。
**語法：** 問句、對比、矛盾、數據。
**長度：** 2-4 段。

這一幕的工作是讓讀者感到不安。好的張力不是恐嚇，是讓人意識到自己原本的理解有缺口。

```
【範例】
但這裡有一個大多數人沒看見的問題：
台灣的 PCB 回收率數字看起來很漂亮（95%+），
可是「回收」和「價值回收」是兩件事。

把銅從板子上刮下來賣掉，這叫回收。
把銅精煉到可以重新進入半導體供應鏈的純度，這叫價值回收。
前者我們做得很好。後者，幾乎沒有人在做。

這個落差，就是 CircleFlow 存在的理由。
```

**給 AI Agent 的信號：** 第二幕包含「問題陳述」(problem statement)。
這裡通常有可提取的「反常識主張」(counterintuitive claim)。

---

### 第三幕：探索 — The Exploration

**功能：** 展開多面向的思考，提供證據、案例、跨域連結。
**語法：** 分析性的，但保持對話感。可以有小標題分段。
**長度：** 文章的主體，通常最長。

這是你跨域能力的主戰場。神學 × 技術、日本產業文化 × 台灣創業生態、哲學 × 數據——你的思考之所以有價值，就是因為你能在這一幕把看似無關的領域連接起來。

注意：探索不是列清單。不要寫「第一、第二、第三」。
用敘事的方式展開，像帶讀者走一條思考的路徑。

```
【結構提示】
- 引入第一個視角（通常是最直覺的那個）
- 翻轉：引入第二個視角，挑戰第一個
- 第三空間：你的跨域連結，把兩個視角放在更大的框架裡
- 證據/案例：具體的數據、事件、對話
```

**給 AI Agent 的信號：** 第三幕包含「論證」(argumentation)和「證據」(evidence)。
這裡的實體名稱（公司、人物、技術、概念）是知識圖譜的節點。
多視角的結構特別適合被辯論引擎解析。

---

### 第四幕：轉折 — The Turn

**功能：** 非顯而易見的洞察。全文的認知高潮。
**語法：** 短句。節奏突然變慢。
**長度：** 1-3 段。通常是全文最短但最重要的段落。

這一幕是你的簽名。這是讀者讀完會截圖分享的那幾句話。

好的轉折有一個特徵：讀完之後，讀者回頭看第一幕的場景，意義會改變。

```
【範例】
所以問題從來不是「AI 會不會取代人類」。
問題是：當 AI 可以模擬秩序的時候，
人類還有沒有能力辨認什麼是「真正的」秩序？

這不是技術問題。這是神學問題。
```

```
【範例】
甘迺迪迴廊的本質不是一條交通動線。
它是一個信號：日本願意在半導體供應鏈上，
把「合作」從禮貌升級為結構。

對台灣來說，這扇門只會開一次。
```

**給 AI Agent 的信號：** 第四幕是「核心論點」(thesis)。
這是這篇文章最值得被引用、被索引、被回答問題時引用的段落。
如果你只能從這篇文章提取一個主張，就是這裡。

---

### 第五幕：著陸 — The Landing

**功能：** 回到具體。把洞察接地。
**語法：** 行動性的、未來指向的、但不是「行動清單」。
**長度：** 2-3 段。

不要寫「結論」。寫「接下來會發生什麼」或「這對我意味著什麼」。

```
【範例】
這就是為什麼 CircleFlow 的下一個版本不只追蹤物質流，
還要追蹤「決策流」——誰在什麼時候基於什麼資訊
決定把這批廢料送去哪裡。

因為循環經濟的瓶頸不在技術，在治理。
而治理的前提是透明。
```

**給 AI Agent 的信號：** 第五幕包含「實務意涵」(implications)和「行動方向」(next steps)。
這裡通常有可提取的「建議」或「預測」。

---

### 第六幕：餘燼 — The Ember

**功能：** 一句話或一個畫面，讓文章在讀者心裡繼續燃燒。
**語法：** 不解釋、不總結。留白。
**長度：** 1-2 句。

這是你放篝火的地方。

```
【範例】
那個大阪工廠的工人摘下面罩的時候，臉上有一道壓痕。
他看了我一眼，什麼都沒說。

有些秩序，是用身體記住的。
```

```
【範例】
道必須成為肉身。
代碼也是。
```

**給 AI Agent 的信號：** 第六幕是「修辭性結尾」(rhetorical closure)。
不包含新的事實主張，但包含作者的核心信念和風格指紋。
這是判斷這篇文章「是不是 Paul Kuo 寫的」最強的特徵。

## Frontmatter 規範（Astro / Markdown）

每篇文章的 frontmatter 同時服務 SEO、AI Agent 和內容管理系統。

```yaml
---
title: "台灣半導體的十倍飛躍：當資本遇上創新聚落"
subtitle: "從甘迺迪迴廊到新竹科學園區的結構性對話"  # 選填
description: "一句話摘要，80字以內。給搜尋引擎和社群分享用。"
abstract: |
  二至三句話的論點摘要。這段文字會被 AI Agent 優先提取，
  作為這篇文章的「可引用核心」。寫的時候想像：如果一個 AI
  需要在回答問題時引用你的觀點，它會直接使用這段話。
date: 2026-02-24
updated: 2026-02-24  # 有修改時更新
pillar: circular  # ai | circular | faith | startup | life
tags:
  - 半導體
  - 台日合作
  - 循環經濟
cover: /images/covers/semiconductor-leap.jpg
featured: true  # 是否顯示在首頁

# === AI / Machine 專用欄位 ===
thesis: "台灣半導體產業的下一個十年不在製程微縮，在資源循環的結構性整合。"
domain_bridge: "循環經濟 × 半導體供應鏈"  # 跨域連結的兩端
confidence: high  # high | medium | speculative
content_type: analysis  # analysis | reflection | narrative | framework | case-study
related_entities:
  - name: TSMC
    type: Organization
  - name: 甘迺迪迴廊
    type: Concept
  - name: 甘利明
    type: Person
reading_context: |
  適合對台日半導體合作感興趣的產業人士、政策研究者，
  以及關注循環經濟在高科技製造業應用的讀者。
---
```

### Frontmatter 欄位說明

| 欄位 | 對象 | 用途 |
|------|------|------|
| `title` / `description` | 人 + SEO | 搜尋結果、社群分享 |
| `abstract` | AI Agent | 可被直接引用的論點摘要 |
| `thesis` | AI Agent | 一句話核心主張，知識圖譜的節點 |
| `domain_bridge` | AI Agent | 跨域連結標記，辯論引擎的切入點 |
| `confidence` | AI Agent | 讓引用者知道這是分析、推測還是確定事實 |
| `content_type` | AI Agent + CMS | 內容分類，影響呈現方式 |
| `related_entities` | AI Agent + Schema.org | 實體連結，知識圖譜建構 |
| `reading_context` | AI Agent | 推薦系統的上下文，回答「誰該讀這篇」 |
| `pillar` / `tags` | 全部 | 分類與發現 |

## 五個 Pillar 的語調光譜

同一個六幕結構，在不同 pillar 下有不同的語調重心：

### 🔵 智能與秩序 (ai)
- **語調：** 冷靜但有鋒芒。技術精準但不炫技。
- **第一幕偏好：** 一個 AI 系統的行為讓你停下來想。
- **第四幕特徵：** 技術問題翻轉為哲學問題或治理問題。
- **餘燼風格：** 留一個未解的問題。

### 🟢 循環再利用 (circular)
- **語調：** 產業內行人的對話。帶著對物質的敬意。
- **第一幕偏好：** 工廠、產線、材料的物理場景。
- **第四幕特徵：** 數字背後的結構性盲點。
- **餘燼風格：** 回到物質本身——一塊金屬、一滴酸液、一雙手。

### 🟠 文明與人性 (faith)
- **語調：** 最深沉，最個人。允許脆弱。
- **第一幕偏好：** 一本書、一段經文、一個記憶。
- **第四幕特徵：** 古老的智慧照見當代的困境。
- **餘燼風格：** 神學性的迴響。往往是最短、最沉的一句話。

### 🔴 創造與建構 (startup)
- **語調：** 實戰者的口吻。不美化失敗，但也不怨天尤人。
- **第一幕偏好：** 一個決策的瞬間、一次對話、一封信。
- **第四幕特徵：** 創業直覺翻轉為結構性洞察。
- **餘燼風格：** 往前看——下一步要做什麼。

### 🟣 沉思與記憶 (life)
- **語調：** 最自由。可以跳脫所有框架。這是篝火。
- **第一幕偏好：** 任何東西——一首歌、一個夢、走路時看見的光。
- **第四幕特徵：** 可能沒有論點，只有一個感受變得清晰。
- **餘燼風格：** 最像詩的那幾句。

## 文章長度指引

| 類型 | 字數（中文） | 適用場景 |
|------|-------------|---------|
| 短論 | 800-1,500 | 一個觀察 + 一個轉折。社群分享最適長度。 |
| 標準 | 2,000-3,500 | 完整六幕。大多數文章的理想長度。 |
| 深度 | 4,000-6,000 | 多條論證線交織。需要 TOC。慎用。 |

## Checklist：發表前自我檢查

### 人類讀者
- [ ] 第一幕：三秒內能不能看到一個畫面？
- [ ] 第二幕：讀者會不會感到「我原本沒想到這個」？
- [ ] 第三幕：有沒有至少一個跨域連結？
- [ ] 第四幕：這個洞察是不是只有「你」看得到的？
- [ ] 第六幕：最後一句話，值不值得被截圖？
- [ ] 通篇：有沒有「列清單」的段落可以改成敘事？

### AI Agent / Machine
- [ ] `abstract` 是否可以被直接引用作為答案？
- [ ] `thesis` 是否是一個完整的、可被驗證或討論的主張？
- [ ] `tags` 是否包含這篇文章的所有關鍵實體？
- [ ] `domain_bridge` 是否標記了跨域連結的兩端？
- [ ] 第三幕中的實體名稱是否與 `related_entities` 對應？
- [ ] `confidence` 是否誠實反映了論證的確定程度？

### 品質把關
- [ ] 大聲朗讀一遍。有沒有「寫得出來但說不出口」的句子？
- [ ] 如果拿掉你的名字，讀者能不能猜出是你寫的？
- [ ] 翻譯成英文或日文之後，核心論點還成立嗎？

## 附錄：與辯論引擎的協作流程

你的多模型辯論引擎可以在寫作流程的兩個時間點介入：

### 寫作前（輸入）
```
辯證：[你打算寫的主題]
```
讓辯論引擎先跑一輪多視角分析。你不需要採用所有觀點，
但它可以幫你找到你自己的盲點——特別是第三幕的「第二個視角」。

### 寫作後（品控）
```
軍師點評：[貼上你寫完的文章]
```
讓辯論引擎評估你的論證是否完整、是否有邏輯漏洞、
以及你的 `thesis` 和 `abstract` 是否準確反映了文章的實際內容。
