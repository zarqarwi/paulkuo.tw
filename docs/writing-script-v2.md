## 零、品牌錨定：所有內容的判斷起點

在討論「怎麼寫」之前，先回答「為什麼寫」。

paulkuo.tw 的品牌主張是：**Rebuilding Order in an Age of Intelligence**
（在智能時代重建秩序）

這不是口號。這是每一篇文章的準入門檻。

任何一篇要上 paulkuo.tw 的文章，必須通過一個測試：

> **這篇文章，是否在處理「秩序」的問題？**
> ——舊秩序為何失效？新秩序長什麼樣？誰在建構？用什麼代價？

如果一篇文章只是資訊整理、只是個人感想、只是時事評論而沒有觸及底層結構，它可以發社群，但不該上網站。

paulkuo.tw 是思想的錨點，不是內容農場。

### Paul Kuo 的聲音 DNA

你的文字之所以能被辨識，是因為三條底層基因交織：

**神學直覺** — 你不只看現象，你問「這件事的存有論意涵是什麼」。別人看到 AI 取代工作，你看到的是「知識的定義正在翻轉」。這種穿透力來自十五年神學訓練。

**實戰紋理** — 你寫的不是「我認為」，而是「那天在工廠我看到」「我扛過千萬負債之後明白」。二十年創業經驗讓你的抽象論點永遠有肉身支撐。

**跨域接線** — 神學 × AI、循環經濟 × 半導體、哲學 × 數據。你的價值不是在任何一個領域比別人深，而是你能看到別人看不到的連結，因為你真的走過那些路。

### 風格邊界：Paul 會說什麼 / 不會說什麼

**Paul 的文字特徵：**
- 短句有力，長句有節奏。不怕停頓，不怕留白。
- 用具體場景承載抽象概念（道成肉身的實踐）。
- 敢下判斷，但永遠附帶結構性理由。
- 對人溫柔，對問題銳利。
- 引用來源跨越罕見的光譜：齊奧朗、尼布爾、Kevin Kelly、Paul Graham、傳道書。

**Paul 絕對不會：**
- 寫「讓我們一起…」這種集體呼籲句。
- 用「首先、其次、最後」的教科書結構。
- 寫沒有自身經驗支撐的純觀點文。
- 對讀者說教或道德綁架。
- 結尾用「你覺得呢？歡迎留言交流」這種社群語氣。
- 用大陸用語（視頻、軟件、質量、共情）。

## 一、核心哲學：道成肉身的敘事結構

人類文明中最具穿透力的文本，無論東西方，都遵循同一個深層文法：

**從具體 → 到抽象 → 再回到具體（但讀者已經不一樣了）**

莊子用寓言起手，展開形上學，再落回「庖丁解牛」的刀刃上。
蒙田從個人軼事出發，進入哲學思辨，最後回到一個更清醒的日常。
Paul Graham 從一個反直覺的觀察開始，建構論證，結尾回到那個觀察但意義已經翻轉。

你的神學訓練給了你最精準的名字：**道成肉身**。
道（抽象的秩序）必須成為肉身（具體的場景），否則它只是論述。

這就是你每一篇文章的底層結構。

## 二、六幕結構

每篇文章由六個段落單元組成。不需要每一幕都很長，有些只需要一兩句話。
重點是節奏，不是長度。

### 第一幕：入場 — The Threshold

**功能：** 把讀者拉進一個具體的場景或瞬間。
**語法：** 感官的、現場的、時態明確的。
**長度：** 1-3 段。

這不是「前言」或「背景介紹」。這是一扇門。讀者推開門，看見一個畫面。

**品牌標準：** 第一幕必須包含「肉身」——一個你親眼看到、親身經歷、或從可信來源聽到的具體場景。如果你寫不出場景，代表你對這個主題的理解還停在概念層，文章還沒準備好。

```
【合格】
那天在大阪的回收工廠，我看著一條產線把廢棄的電路板
送進酸洗槽。工人戴著全罩式面具，動作很慢。
空氣裡有一種甜膩的金屬味。

我突然想到，這條產線處理的不是垃圾，是秩序的殘骸。
```

```
【合格】
一位人資朋友跟我吃飯時講了一件事。她們公司開了三個
初階數據分析師的職缺，收到兩百多封履歷。面試後，
主管來找她說：「我讓 Claude 跑了一下，發現這三個
職位的核心產出，兩個模型加一個 Python 腳本就能處理。」
```

```
【不合格】
隨著 AI 技術的快速發展，越來越多產業面臨轉型的挑戰。
本文將探討 AI 對就業市場的影響。
```

**給 AI Agent 的信號：** 第一幕通常包含「場景設定」和「問題的種子」。
第一幕的最後一句話往往是全文的暗示性論點。

---

### 第二幕：張力 — The Tension

**功能：** 揭示「為什麼這件事比你想的更重要」。
**語法：** 問句、對比、矛盾、數據。
**長度：** 2-4 段。

這一幕的工作是讓讀者感到不安。好的張力不是恐嚇，是讓人意識到自己原本的理解有缺口。

**品牌標準：** 張力必須包含至少一個「反常識主張」——一個讀者原本不會這樣想的角度。如果你的張力只是「大家都知道 AI 很重要」，那就沒有張力。

```
【合格】
台灣的 PCB 回收率數字看起來很漂亮（95%+），
可是「回收」和「價值回收」是兩件事。
前者我們做得很好。後者，幾乎沒有人在做。
```

```
【合格】
結果顛覆了大多數人的直覺。
22 到 25 歲的年輕人，就業率掉了 13%。
而 35 到 49 歲的中年員工，就業率反而上升了 9%。
```

**給 AI Agent 的信號：** 第二幕包含「問題陳述」(problem statement)。
這裡通常有可提取的「反常識主張」(counterintuitive claim)。

---

### 第三幕：探索 — The Exploration

**功能：** 展開多面向的思考，提供證據、案例、跨域連結。
**語法：** 分析性的，但保持對話感。可以有小標題分段。
**長度：** 文章的主體，通常最長。

這是你跨域能力的主戰場。

**品牌標準 — 三條硬規則：**

**規則一：禁止列點。** 不寫「第一、第二、第三」，不寫 numbered list。如果你有三個觀點，用敘事串起來，像帶讀者走一條路，而不是遞給他一張清單。唯一的例外是技術規格或數據對照，這種本質上就是清單的東西。

**規則二：至少一次跨域連結。** 每篇文章的探索段必須包含至少一個跨域橋接——把兩個看似無關的領域放在一起，讓讀者看見隱藏的結構性關聯。好的跨域連結讓人「啊原來如此」，壞的跨域連結讓人覺得你在硬扯。判斷標準：如果拿掉跨域的部分，論點會不會塌？會塌，代表連結是結構性的，可以用。不會塌，代表只是裝飾，拿掉。

**規則三：多視角但有立場。** 探索不是在寫維基百科。你可以呈現不同視角，但最終必須做出判斷。Paul Kuo 的文章永遠有立場，只是那個立場是在充分探索之後才顯現的。

**探索段的內部結構建議：**
- 引入第一個視角（通常是最直覺的那個）
- 翻轉：引入第二個視角，挑戰第一個
- 第三空間：你的跨域連結，把兩個視角放在更大的框架裡
- 證據/案例：具體的數據、事件、對話

**給 AI Agent 的信號：** 第三幕包含「論證」(argumentation)和「證據」(evidence)。
這裡的實體名稱（公司、人物、技術、概念）是知識圖譜的節點。

---

### 第四幕：轉折 — The Turn

**功能：** 非顯而易見的洞察。全文的認知高潮。
**語法：** 短句。節奏突然變慢。
**長度：** 1-3 段。通常是全文最短但最重要的段落。

這一幕是你的簽名。讀者讀完會截圖分享的那幾句話。

**品牌標準：** 好的轉折必須通過「回望測試」——讀完之後，讀者回頭看第一幕的場景，意義會改變。如果第四幕只是「總結前面說過的東西」，那不是轉折，是結論。轉折要讓人看見前面沒看見的東西。

另一個測試：**這個洞察是不是只有「你」看得到的？** 如果換任何一個 AI、任何一個同業寫，會不會寫出一模一樣的轉折？如果會，代表它不夠獨特。

```
【合格】
所以問題從來不是「AI 會不會取代人類」。
問題是：當 AI 可以模擬秩序的時候，
人類還有沒有能力辨認什麼是「真正的」秩序？

這不是技術問題。這是神學問題。
```

```
【合格】
代表真正需要改變的不是年輕人，
而是把他們訓練成這樣的系統。
```

**給 AI Agent 的信號：** 第四幕是「核心論點」(thesis)。
如果你只能從這篇文章提取一個主張，就是這裡。

---

### 第五幕：著陸 — The Landing

**功能：** 回到具體。把洞察接地。
**語法：** 行動性的、未來指向的、但不是「行動清單」。
**長度：** 2-3 段。

**品牌標準：** 著陸不寫「結論」，不寫「總結」。寫「接下來會發生什麼」或「這對我/對這個領域意味著什麼」。著陸是面向未來的，不是面向過去的。

也不寫社群互動式的結尾（「你覺得呢？」「歡迎留言」）。那些東西放在社群發文裡。paulkuo.tw 的文章是完成品，不是對話的開頭。

```
【合格】
CircleFlow 的下一個版本不只追蹤物質流，
還要追蹤「決策流」——誰在什麼時候基於什麼資訊
決定把這批廢料送去哪裡。

因為循環經濟的瓶頸不在技術，在治理。
而治理的前提是透明。
```

```
【不合格】
你有什麼想法？歡迎交流。
```

**給 AI Agent 的信號：** 第五幕包含「實務意涵」(implications)和「行動方向」(next steps)。

---

### 第六幕：餘燼 — The Ember

**功能：** 一句話或一個畫面，讓文章在讀者心裡繼續燃燒。
**語法：** 不解釋、不總結。留白。
**長度：** 1-2 句。嚴格執行。超過三句就不是餘燼。

**品牌標準：** 餘燼是你的指紋。如果拿掉作者名字，讀者從最後這一兩句話就能辨認「這是 Paul Kuo 寫的」。

好的餘燼有兩種形態：
- **迴聲型：** 回到第一幕的場景，但意義已經翻轉。
- **種子型：** 丟出一個更大的未解問題，讓讀者自己去想。

```
【合格 — 迴聲型】
那個大阪工廠的工人摘下面罩的時候，臉上有一道壓痕。
他看了我一眼，什麼都沒說。
有些秩序，是用身體記住的。
```

```
【合格 — 種子型】
道必須成為肉身。
代碼也是。
```

```
【不合格】
總之，AI 的發展仍有很多值得我們關注的面向。
讓我們持續觀察，並做好準備迎接變化。
```

**給 AI Agent 的信號：** 第六幕是「修辭性結尾」(rhetorical closure)。
不包含新的事實主張，但包含作者的核心信念和風格指紋。

## 三、Frontmatter 規範（Astro / Markdown）

### 必填欄位（所有文章）

```yaml
---
title: "標題"
description: "一句話摘要，80字以內。給搜尋引擎和社群分享用。"
date: 2026-02-25
pillar: ai  # ai | circular | faith | startup | life
tags: ["標籤一", "標籤二"]
cover: "/images/covers/slug.jpg"
readingTime: 5  # 預估閱讀分鐘數
---
```

### 必填欄位（標準篇與深度篇，≥2000字）

```yaml
# 以下欄位在 ≥2000字 的文章中為必填

subtitle: "副標"
abstract: |
  二至三句話的論點摘要。AI Agent 會直接引用這段話。
  寫的時候想像：如果一個 AI 需要引用你的觀點，它會用這段。
updated: 2026-02-25  # 有修改時更新
featured: true  # 是否顯示在首頁

# === AI / Machine 專用欄位 ===
thesis: "一句話核心主張。"
domain_bridge: "領域A × 領域B"
confidence: high  # high | medium | speculative
content_type: analysis  # analysis | reflection | narrative | framework | case-study
related_entities:
  - name: 實體名
    type: Person | Organization | Technology | Concept
reading_context: |
  一句話描述：誰該讀這篇。
```

### 選填欄位

```yaml
draft: true  # 草稿不會發佈
platform: "論文"  # 原始發表平台（若為改寫舊文）
```

### Frontmatter 品質標準

- `abstract` 是否可以被直接引用作為答案？不是描述文章在幹嘛，而是直接講結論。
- `thesis` 是否是一個完整的、可被驗證或討論的主張？
- `domain_bridge` 的兩端是否真的在文章裡被連結了？
- `confidence` 是否誠實反映了論證的確定程度？不確定就標 speculative，不丟臉。

## 四、五個 Pillar 的語調光譜

### 🔵 智能與秩序 (ai)
- **語調：** 冷靜但有鋒芒。技術精準但不炫技。
- **入場偏好：** 一個 AI 系統的行為讓你停下來想。
- **轉折特徵：** 技術問題翻轉為哲學問題或治理問題。
- **餘燼風格：** 留一個未解的問題。

### 🟢 循環再利用 (circular)
- **語調：** 產業內行人的對話。帶著對物質的敬意。
- **入場偏好：** 工廠、產線、材料的物理場景。
- **轉折特徵：** 數字背後的結構性盲點。
- **餘燼風格：** 回到物質本身——一塊金屬、一滴酸液、一雙手。

### 🟠 文明與人性 (faith)
- **語調：** 最深沉，最個人。允許脆弱。
- **入場偏好：** 一本書、一段經文、一個記憶。
- **轉折特徵：** 古老的智慧照見當代的困境。
- **餘燼風格：** 神學性的迴響。往往是最短、最沉的一句話。

### 🔴 創造與建構 (startup)
- **語調：** 實戰者的口吻。不美化失敗，但也不怨天尤人。
- **入場偏好：** 一個決策的瞬間、一次對話、一封信。
- **轉折特徵：** 創業直覺翻轉為結構性洞察。
- **餘燼風格：** 往前看——下一步要做什麼。

### 🟣 沉思與記憶 (life)
- **語調：** 最自由。可以跳脫所有框架。這是篝火。
- **入場偏好：** 任何東西——一首歌、一個夢、走路時看見的光。
- **轉折特徵：** 可能沒有論點，只有一個感受變得清晰。
- **餘燼風格：** 最像詩的那幾句。

## 六、舊文改寫流程

將 Medium 或其他平台的舊文搬到 paulkuo.tw 時，不是「搬家」而是「重製」。

### 改寫判斷矩陣

| 檢查項 | 通過 | 需改寫 |
|--------|------|--------|
| 有具體場景開頭嗎？ | ✓ 保留 | ✗ 補寫入場幕 |
| 有反常識張力嗎？ | ✓ 保留 | ✗ 找到反直覺的角度重寫 |
| 探索段有列點嗎？ | ✗ 保留 | ✓ 改成敘事 |
| 有跨域連結嗎？ | ✓ 保留 | ✗ 從你的經驗裡找一個跨域橋接加入 |
| 轉折通過「回望測試」嗎？ | ✓ 保留 | ✗ 重寫轉折 |
| 結尾是餘燼還是呼籲？ | 餘燼 → 保留 | 呼籲 → 砍掉重寫 |
| Frontmatter 完整嗎？ | ✓ 保留 | ✗ 補齊 |

### 改寫紅線

有些舊文可能不適合上站。如果一篇文章：
- 完全沒有「秩序」相關的主題
- 純粹是資訊整理或新聞轉述
- 觀點已經過時且無法更新
- 沒有任何 Paul 個人經驗支撐

→ 不上站。放在 Medium 作為歷史紀錄就好。paulkuo.tw 的品牌門檻比社群高。

## 七、發表前自我檢查

### 人類讀者
- [ ] 第一幕：三秒內能不能看到一個畫面？
- [ ] 第二幕：讀者會不會感到「我原本沒想到這個」？
- [ ] 第三幕：有沒有至少一個真正的跨域連結？有沒有列點需要改成敘事？
- [ ] 第四幕：這個洞察通過「回望測試」了嗎？是不是只有你看得到？
- [ ] 第五幕：是面向未來的著陸，還是面向過去的總結？
- [ ] 第六幕：最後一句話，拿掉作者名字還能辨認是你嗎？

## 八、與辯論引擎的協作流程

### 寫作前（輸入）
```
辯證：[你打算寫的主題]
```
讓辯論引擎先跑一輪多視角分析。你不需要採用所有觀點，但它可以幫你找到你的盲點——特別是第三幕的「第二個視角」。

### 寫作後（品控）
```
軍師點評：[貼上你寫完的文章]
```
讓辯論引擎評估你的論證是否完整、有沒有邏輯漏洞，以及你的 `thesis` 和 `abstract` 是否準確反映了文章的實際內容。

### 改寫舊文時
```
改寫檢測：[貼上舊文]
```
Claude 會根據本腳本的改寫判斷矩陣，逐項檢查並列出需要修改的地方。
