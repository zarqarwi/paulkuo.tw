作業の途中でレート制限に引っかかる——Claude ユーザーにとって一番イラッとする瞬間だろう。

以前の自分のやり方は、ブロックされてからやっと Settings ページを見に行くこと。そこで追加クレジットがすでに 84%、7日間の使用量も半分を超えていたことに気づく。この数字がずっと画面に出ていればいいのに、とずっと思っていた。

そこでこの問題を解決する Chrome Extension を作った。「Claude Usage Nyan」という名前で、ツールバーにオレンジのトラ猫が座っている。ワンクリックでどれだけ使ったかがわかる。

## 何ができるか

インストールすると、三箇所で使用量が確認できる。

ツールバーのオレンジ猫アイコンには小さなバッジが表示され、4秒ごとに自動切り替え——5時間セッションの使用量、7日間の使用量、追加クレジットの残り、リアルタイムの費用。緑は安全、黄色は注意、オレンジはもうすぐ、赤はほぼ上限。何もクリックしなくても、一目でわかる。

ポップアップをクリックすると完全な使用量カードが表示される。上半分は Anthropic 公式の数字：5時間セッション百分率、7日間使用量百分率、追加使用量の使用済みと上限、それぞれリセットまでのカウントダウン付き。下半分はリアルタイムのトークン追跡：直前の会話で何 input token、何 output token 使ったか、いくらかかったか、どのモデルか、すべて一覧表示。

claude.ai ページの右下には半透明のフローティングステータスバーもあり、常に使用量の概要を表示する。猫の頭をクリックすると折りたためる。

## 二本のパイプライン、差異を見る

このツールが他のトラッカーと最も違う点は、二本のデータパイプラインを同時に走らせることだ。

一本目は公式。Extension が5分ごとに claude.ai の usage API を呼び出し、返ってくる数字は Settings ページで自分で見るのとまったく同じ。この数字は権威的だが、遅延がある——Anthropic 側の更新はリアルタイムではなく、明らかに多く使っていてもパーセンテージが動かないことがある。

二本目はリアルタイム推算。Extension は claude.ai ページ上で Claude との会話の API コールをすべてインターセプトし、リクエスト送信時に input token を推算、レスポンスのストリーミング中に output token を累積し、モデル価格から費用を計算する。これはリアルタイムだが推算値であり、公式の数字とは誤差がある。

両方を並べて置くと、公式のパーセンテージと実際に消費したトークンがどれだけ違うか自分で観察できる。この差異を可視化することが、このツールを作った目的の一つだ。

## 作る前にまず一通り調べた

コードを書き始める前に、市場にある既存ツールを調査した。

結果、この種の小さなツールはすでにたくさんあり、見つけただけで十個以上。しかしほぼすべてが macOS のネイティブアプリで、.dmg をダウンロードしてインストールする形式。Windows と Linux ユーザーには選択肢がない。そして大半は公式使用量追跡かリアルタイムトークン計算のどちらか一方だけで、両方を組み合わせたものはない。インターフェースはすべて英語で、アジア市場向けのローカライゼーションは皆無。

なのでポジショニングは明確になった：Chrome Extension にしてクロスプラットフォーム対応、二本のパイプラインを同時に走らせ、中国語ファースト。後から英語と日本語のインターフェースも追加し、Chrome がブラウザの言語に応じて自動切り替えする。

## 開発中にハマったいくつかの落とし穴（技術に興味がなければスキップ可）

開発全体を Claude との対話で進めたが、順風満帆ではなかった。

最初の落とし穴は API フォーマット。Anthropic の usage API には公開ドキュメントがなく、フォーマットを推測するしかなかった。初めて接続した時、ポップアップに生の JSON がどっと表示された。しかしこの JSON の塊自体が答えだった——five_hour.utilization や seven_day.resets_at といったフィールド名を見た瞬間、パースの仕方がわかった。だからポップアップにデバッグモードを意図的に残した：パース失敗時は生の JSON を直接表示する。将来 API フォーマットが変わっても素早く対応できる。

二つ目の落とし穴はもっと面白かった。Chrome Extension の content script は isolated world という隔離環境で動く。そこで window.fetch をパッチして claude.ai の API コールをインターセプトしようとしたが、何もキャッチできなかった。しばらくして判明：isolated world の window とページ自体の window は別のオブジェクトだ。解決策は Chrome MV3 の world: "MAIN" 設定を使い、インターセプトスクリプトをページのコンテキストに直接注入し、CustomEvent でデータを isolated world のブリッジレイヤーに渡すこと。一つの問題を二層に分けて解決した。

三つ目の落とし穴はアイコン。三回イテレーションした——一版目は平凡すぎ、二版目は円の中に押し込んで猫に見えず、三版目でオレンジのトラ猫の参考写真を送り、「耳が外に開いていて、額に M 字の模様」と指定して、ようやくしっくりきた。スペックで表現しにくいことがある。参考画像一枚は百の言葉に勝る。

## 制限事項を先に言っておく

Anthropic の usage API には公開ドキュメントがなく、フォーマットはいつでも変わりうる。変わればこの Extension も更新が必要で、メンテナンスがなければ壊れる。リアルタイムトークンは推算値であり、正確な数字ではない。英語は約4文字で1トークン、中国語は約1.5文字で1トークン——どちらも Anthropic 実際の tokenizer とは差がある。

追跡できるのは claude.ai ウェブ版のみ。Claude Code CLI を使っている場合は別のチャネルを通るため、この Extension ではキャプチャできない。

Extension は API アクセスのために claude.ai のセッション Cookie を読む必要がある。すべてのデータはブラウザのローカルにのみ保存され、外部サーバーには一切送信されない。完全オープンソース。ただし、どんな Extension をインストールする前でも、自分でリスクを判断するのは基本だ。

---

インストールは簡単：GitHub リポジトリを clone、Chrome の開発者モードを有効化、フォルダを読み込み——三ステップ。繁体中国語・英語・日本語のインターフェースに対応。
