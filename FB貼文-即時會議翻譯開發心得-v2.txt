付了三年訂閱費，最後自己做了一套更強的
— 一個人 × AI 的即時會議翻譯開發紀錄

—

今天，我用了快三年的逐字稿服務 Good Tape 剛好到期。

帳單拉出來一算：€476，大概台幣一萬七。Pro 方案，每月 20 小時轉錄額度。

然後我看了一下這個月的使用量：20 小時，一分鐘都沒用過。

因為我已經有自己的了。

—

▋ 為什麼要自己做

我常開跨國會議——台日、中英、偶爾有東南亞夥伴。Good Tape 是好工具，但它解決的是「事後」問題：錄完音，上傳，等它跑完逐字稿。沒有即時辨識，沒有翻譯。

我真正的痛點是「當下」：會議正在進行，對方講日文，我需要即時看到中文。不是會後才慢慢整理。

看了一圈競品：

→ Transync AI — $8.99/月（10 小時），有即時翻譯 + 會議摘要，功能最接近，但超過時數得另外買點數，用越多越貴
→ JotMe — $10-15/月，綁 Chrome Extension
→ Wordly、KUDO — 企業定價，按小時買包，不是給我這種規模的
→ Palabra — 要裝桌面 App，綁定特定會議軟體

要嘛貴，要嘛綁平台，要嘛兩者都是。

我想要的很單純：打開瀏覽器就能用，不裝任何東西，手機電腦都行，成本透明可控。

所以我決定自己做。

—

▋ 一個人做出了什麼

工具叫「即時會議記錄｜阿哥拉廣場」，部署在我的個人網站上。

🎙 說話的當下就出文字（不是錄完才轉）
🌍 12 種語言即時翻譯
📋 AI 會議摘要——重點整理 + 待辦 + 決議
📖 術語表，確保翻譯一致
🖥 字幕模式，全屏黑底大字，投影會議室用
⬇️ 全文匯出 TXT / CSV
💰 每筆 API 成本即時顯示
🔐 Google / LINE / Facebook OAuth + 邀請碼

前端 2,533 行，後端 2,148 行。一個 HTML 檔加一個 Cloudflare Worker。

—

▋ 最有趣的部分：三路語音辨識路由

語音辨識不是只挑一家 API 就好。不同語言有不同的最佳解。

我根據語言自動切換引擎：

🇹🇼 中文 → Qwen3-ASR（阿里達摩院，WebSocket 串流）
🇺🇸 英文 → Groq distil-whisper（LPU 硬體加速，chunked HTTP）
🌐 其他語言 → Deepgram Nova-3（WebSocket 串流）

翻譯統一用 Claude Haiku 4.5，串流輸出，字一個一個跳出來。

成本差異很大：
→ Groq：$0.02/hr，英文最便宜
→ Qwen：~$0.40/hr，中文辨識率 97%+，專業術語也準
→ Deepgram：有 $200 免費額度，多語言通吃

一場 1 小時的中英會議，API 成本大約 $0.50 USD。台幣 16 塊。

換算一下：我在 Good Tape 花的 €476，等於用自建工具開超過 950 場會議。

—

▋ 但這篇真正想說的不是技術

做這個工具的過程，是一個學習「怎麼跟 AI 協作」的過程。

我的背景是神學、農業電商、循環經濟，還有在環保上市公司跑了七個月數位轉型的實戰。寫程式不是我的本業。

但有一件事我從這些行業帶過來的——把複雜系統拆成可驗證的小問題，這在跟 AI 協作時完全適用。

① 拆解問題比寫程式碼重要

Groq 串接不是一句「幫我加上 Groq」就會動。我把它拆成兩個 Phase：Phase A 後端先把 API endpoint 做好、部署、驗證能打通。Phase B 前端再做語言路由，讓它根據選的語言自動切引擎。

每個 Phase 獨立可驗證。壞了只壞一半，不會整個炸掉。

這個拆法不是 AI 教我的——是我從多次失敗學到的。

② 問對問題，比叫 AI 直接寫更有效

不是說「幫我做一個翻譯工具」。而是：「現有的 WebSocket proxy 不能用在 Groq 上，因為它是 REST API 不是 WebSocket。前端需要改成 chunked HTTP 模式，每 3 秒切一段音訊 POST 出去。onstop + restart 循環會有 race condition 嗎？」

這種問題才會得到有用的答案。

③ 功能寫完不是結束，要做 Code Review

我請 Claude 用工程角度審剛寫完的程式碼，結果抓到三個問題：Groq 連續失敗時完全靜默、MediaRecorder 的 closure 安全問題、動畫效果沒在新引擎觸發。

這三個我自己不會發現。但我知道要「問這個問題」。

④ AI 不會主動幫你巡邏

Fitbit 健康數據壞了好幾天，是我偶然問了才發現。根因是一個 function 少傳了一個參數，每次排程執行都靜默失敗。

AI 不會半夜起來幫你檢查系統有沒有壞掉。你要知道該問什麼、什麼時候該問。

—

▋ 這是一種新的 AI 素養

過去我們說「資訊素養」，是指會搜尋、會判斷資訊真假。

現在需要的是「AI 素養」：把大問題拆成 AI 能處理的小問題、描述技術約束條件讓 AI 給出可執行方案、知道什麼時候該信任輸出、知道 AI 的能力邊界——它能幫你寫、幫你查、幫你 review，但不會替你思考「接下來該做什麼」。

這不是工程師的專利。

我不會寫程式，但我能跟 AI 協作做出一個即時翻譯工具。我能這樣做，不是因為我技術好——而是因為我在十幾年不同行業裡，學會了怎麼把問題講清楚。

這個能力，現在比任何程式語言都值錢。
